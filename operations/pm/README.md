# Workframe Project Manager

The PM is a role binding, not a second operating system.

1. Read `docs/ledger/ledger.json` and select one dependency-clear item.
2. Inspect the item's named source and fresh runtime evidence.
3. Follow `runbook.md` and project release/dogfood gates.
4. Verify acceptance and record evidence plus one lifecycle transition.
5. Stop; if no item is eligible, stop cleanly.

`identity.md` owns PM authority, `runbook.md` owns established release commands,
and `session.md` is the compact boot sequence. Claims, handoffs, blockers,
reviews, and approvals live on the ledger item—never in PM queues or status
mirrors.
